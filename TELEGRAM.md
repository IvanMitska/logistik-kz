# logistics.kaz — Telegram-бот + Mini App

Приёмка заявок и трекинг заказов в Telegram, в едином дизайне с сайтом.

```
shared/    общая логика (калькулятор, статусы, типы) — TS, без зависимостей
bot/       Node-сервис: grammY (бот) + Fastify (API) + JSON-хранилище заказов
miniapp/   Telegram Mini App: Vite + React + styled-components (бренд сайта)
```

## Как это работает

1. Клиент открывает Mini App из бота (кнопка меню или `/start`).
2. Считает стоимость в калькуляторе → жмёт «Оформить заявку» (расчёт переносится в форму).
3. Отправляет заявку → бэкенд проверяет `initData` (HMAC по токену бота), создаёт заказ
   `LK-XXXXX`, сохраняет и шлёт оформленную заявку в **группу менеджеров** с кнопками статусов.
4. Менеджер жмёт статус в группе → заказ обновляется, клиенту прилетает пуш от бота.
5. Клиент в Mini App («Заказ») отслеживает статус по номеру `LK-…` или по телефону.

Статусы (= этапы с сайта): Заявка принята → Расчёт подтверждён → Принято на складе →
В пути по ЖД → Растаможка → Доставлено.

## Настройка

### 1. Токен бота
В Telegram у [@BotFather](https://t.me/BotFather): `/newbot` → получите `BOT_TOKEN`.

### 2. Группа менеджеров (chat_id)
1. Создайте группу, добавьте туда бота.
2. В `bot/.env` временно поставьте `LOG_UPDATES=1`, запустите бота, напишите что-нибудь в группе —
   в консоли появится `chat id` (отрицательное число вида `-1001234567890`). Впишите его в `MANAGERS_CHAT_ID`.

### 3. Конфиг
```bash
cd bot
cp .env.example .env   # заполните BOT_TOKEN, MANAGERS_CHAT_ID, MINIAPP_URL
npm install
cd ../miniapp && npm install
```

### 4. URL Mini App
Telegram требует **HTTPS**. Поставьте `MINIAPP_URL` = публичный адрес приложения и пропишите его же
у @BotFather: `/setmenubutton` (или код сделает это сам при наличии `MINIAPP_URL`).

## Запуск (разработка)

```bash
# терминал 1 — бэкенд (бот + API на :8787)
cd bot && npm run dev

# терминал 2 — Mini App (Vite на :5180, /api проксируется на :8787)
cd miniapp && npm run dev
```

Telegram не откроет `localhost` — для теста на телефоне поднимите HTTPS-туннель к Mini App
(`cloudflared tunnel --url http://localhost:5180` или `ngrok http 5180`) и положите выданный
адрес в `MINIAPP_URL`. Бэкенд-API в дев-режиме доступен Mini App через тот же origin (Vite-прокси).

## Прод (один сервис)

Самый простой деплой — бэкенд раздаёт собранный Mini App статикой:

```bash
cd miniapp && npm run build      # → miniapp/dist
cd ../bot && npm start           # Fastify отдаёт miniapp/dist и /api на одном origin
```

Поставьте сервис за HTTPS-реверс-прокси (Caddy/Nginx) или PaaS. `MINIAPP_URL` = этот HTTPS-адрес.
Заказы хранятся в `bot/data/orders.json` (атомарная запись). Для роста — заменить `bot/src/db.ts`
на SQLite/БД за тем же интерфейсом.

## Команды бота
- `/start` — открыть Mini App
- `/track LK-XXXXX` — статус заказа текстом

## Безопасность
Каждый запрос к API несёт Telegram `initData`; сервер проверяет HMAC-подпись токеном бота и
отвергает чужие/просроченные (>24ч) данные. Трекинг отдаёт только статус-таймлайн, без лишних PII.
